# 路演 PPT 大纲

> 演示时长：5-10 分钟 | 建议页数：15-20 页

---

## 建议 PPT 结构

### 第一部分：项目概述（2-3 页）
1. **封面** — 项目名称 / 团队 / 日期
2. **项目背景与痛点** — 校友成果分散管理 / 缺乏展示平台
3. **解决方案概览** — 填报→审核→大屏展示 全流程闭环

### 第二部分：系统演示（6-8 页）
4. **整体架构图** — 学生H5 / 管理后台 / Unity大屏 / Spring Boot 后端
5. **学生端 H5 演示** — 资料填报 / 人脸录入 / 进度查询
6. **管理后台演示** — Dashboard / 三级审核 / 资料库管理
7. **Unity 大屏演示（重点）** — 4种展示模式切换
8. **人脸识别演示** — 摄像头抓帧 → 特征提取 → 比对 → 个性化展示
9. **WebSocket 实时推送** — 管理后台控制大屏实时响应
10. **Docker 一键部署** — docker compose up -d

### 第三部分：技术亮点（3-4 页）
11. **技术架构** — Spring Boot + Vue3 + Unity + ArcFace
12. **AI 赋能开发** — Vibe Coding / Claude Code / 16次迭代
13. **性能与安全** — 人脸缓存层 / SQL优化 / Nginx安全加固
14. **关键技术决策** — Mock模式 / ArcFace离线SDK / WebSocket保活

### 第四部分：总结展望（2-3 页）
15. **功能完整性对照** — 对照需求逐项展示完成度
16. **迭代历程回顾** — 4阶段16次迭代时间线
17. **未来展望** — 多校区部署 / 移动端适配 / AI推荐
18. **致谢 / Q&A**

---

## 演示录屏建议流程（5-10 分钟）

| 时间段 | 内容 | 时长 |
|--------|------|------|
| 0:00-1:30 | 项目背景 + 架构介绍 | 1.5 min |
| 1:30-2:30 | 学生端 H5：登录→填报→人脸录入 | 1 min |
| 2:30-3:30 | 管理后台：审核流程→上架→Dashboard | 1 min |
| 3:30-6:00 | **Unity 大屏（重点）**：轮播→列表→搜索→人脸识别触发→个人模式 | 2.5 min |
| 6:00-7:00 | WebSocket 实时推送演示 | 1 min |
| 7:00-8:00 | 技术亮点 + AI 赋能 + 迭代回顾 | 1 min |
| 8:00-9:00 | 总结 + 未来展望 | 1 min |

## 录屏工具推荐
- **OBS Studio**（免费开源，Windows/Mac/Linux）
- Windows 自带 Xbox Game Bar（Win+G）
- 录屏分辨率建议 1080p+，清晰展示界面细节
