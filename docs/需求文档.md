# 校友成果展览系统 — 需求文档

> 版本：v1.0 | 更新日期：2026-06-16

---

## 一、项目背景

高校校友在学习、工作、生活中积累了丰富的优秀成果（荣誉奖项、作品、成绩等），这些成果散落在各个渠道，缺乏统一的管理和展示平台。本项目旨在构建一套 **"校友填报 → 三级审核 → 大屏人脸识别个性化展示"** 的全流程管理系统，用于校内展览馆/成果展示厅等场景。

## 二、核心用户角色

| 角色 | 职责 |
|------|------|
| 学生 / 校友 | 填报个人成果资料，录入人脸信息，查看审核进度 |
| 院级审核员 | 第一级审核：审核本院学生提交的资料 |
| 教务处 | 第二级审核：复审院审通过的资料 |
| 校级管理员 | 上架/下架管理，大屏控制，用户/标签管理，数据统计 |

## 三、功能需求

### 3.1 学生端 H5（Vue3 + Vant）

| 功能模块 | 详细需求 |
|----------|----------|
| 认证登录 | 账号密码登录，Sa-Token 会话管理 |
| 资料填报 | 选择成果类别（荣誉/作品/成绩/其他），填写标题/描述/时间，上传图片/视频/文档 |
| 资料管理 | 查看我的资料列表（草稿/审核中/已发布/驳回），编辑/删除草稿或驳回资料 |
| 人脸录入 | 调用摄像头拍照或上传照片，注册人脸特征 |
| 进度查询 | 查看审核状态和审核意见 |

### 3.2 管理后台 Web（Vue3 + Element Plus）

| 功能模块 | 详细需求 |
|----------|----------|
| Dashboard | 资料总数、审核进度统计、学院/类别分布、近7天识别趋势 |
| 审核管理 | 院级审核（一级）、教务处审核（二级），通过/驳回+审核意见 |
| 资料库管理 | 查看全部已发布资料，设置上架/下架状态，调整展示权重，设置标签 |
| 用户管理 | 用户列表、新增/编辑用户、重置密码、启用/禁用 |
| 标签管理 | 标签的增删改查，用于大屏分类展示 |
| 大屏控制 | 强制推送指定资料到大屏、发送通知、切换模式、查看在线状态 |

### 3.3 Unity 大屏端（Unity 2022.3 URP）

| 功能模块 | 详细需求 |
|----------|----------|
| 默认轮播（Carousel） | 双卡片交叉切换，3D翻转动画，KenBurns缓慢缩放，Bloom后处理 |
| 列表浏览（List） | 网格瀑布流布局，按类别筛选，滚动加载 |
| 关键词搜索（Search） | 输入关键词实时搜索 |
| 人脸识别（Personal） | 摄像头500ms抓帧→人脸检测→特征提取→后端比对→命中后镜头推近+欢迎动画+专属轮播 |
| WebSocket 实时推送 | 25s心跳+5s断线重连，响应管理后台推送指令 |
| 粒子特效 | 转场粒子系统、欢迎粒子特效 |

### 3.4 后端 API（Spring Boot 3.2）

| 功能模块 | 详细需求 |
|----------|----------|
| 认证授权 | Sa-Token 登录/登出，角色权限控制（5角色） |
| 资料 CRUD | 资料创建/编辑/删除/提交/查询，草稿状态管理 |
| 三级审核流 | 院审→教务审→已发布，驳回可重新提交 |
| 文件存储 | MinIO 图片/视频/文档上传，匿名读策略 |
| 人脸比对 | 接收Unity端1032字节特征向量，余弦相似度比对 |
| WebSocket | 大屏推送通信，心跳保活 |
| 数据统计 | Dashboard 概览、分布统计、趋势分析 |

## 四、非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 人脸比对<200ms，API响应<500ms，4K 60fps大屏渲染 |
| 安全 | 密码加密存储、人脸特征加密、参数校验、Nginx安全头+速率限制 |
| 可用性 | WebSocket断线自动重连、异常降级处理、夜间定时重启建议 |
| 兼容性 | 移动端H5适配、4K大屏分辨率、跨浏览器 |

## 五、技术约束

- 人脸识别：虹软 ArcFace 4.x 离线SDK（C++ DLL → C# DllImport），Demo 阶段支持 Mock 模式
- 开发模式：Vibe Coding — 需求清晰、分段实现、快速迭代
- 部署：Docker Compose 一键编排
